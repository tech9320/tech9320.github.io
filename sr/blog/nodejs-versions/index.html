Node.js - najnovije verzije i feature-i

Node.js je popularno JavaScript runtime okruženje koje omogućava programerima da grade skalabilne aplikacije visokih performansi. Sa čestim ažuriranjima, Node.js nastavlja da poboljšava performanse i karakteristike. Najnovije verzije, verzije 16, 18 i 20, donele su mnogo uzbudljivih feature-a koje programeri mogu da istraže. U ovom blogu ćemo pogledati neke od novih feature-a i zašto je neophodno pratiti nadogradnje Node.js-a.

Počećemo sa Node.js verzijom 16 i feature-ima koje je on doneo.

Node.js Version 16

Node.js 16 je LTS (Long Term Support) izdanje, što znači da će dobijati ažuriranja i bezbednosne ispravke u naredne tri godine. Neke od značajnijih feature-a uključuju:

V8 version 9.0

Node.js verzija 16 uključuje najnoviju verziju V8 (verzija 9.0), koja donosi nekoliko poboljšanja performansi i novih jezičkih funkcija kao što su opciono ulančavanje (optional chaining) i nulto spajanje (nullish coalescing).

Primer pre uvođenja Node.js 16:

    // Bez korišćenja optional chaining operatora
    const user = {
    name: 'Example Name',
    address: {
        city: 'Belgrade',
        state: 'Serbia'
    }
    };

    const state = user.address && user.address.state;
    console.log(state); // Izlaz: "Serbia"

Nakon uvođenja Node.js 16:

    // Korišćenjem optional chaining operatora
    const user = {
    name: 'Example Name',
    address: {
        city: 'Belgrade',
        state: 'Serbia'
    }
    };

    const state = user.address?.state;
    console.log(state); // Izlaz: "Serbia"

Drugi primer je predstavljen korišćenjem funkcije stepenovanja.

    // Pre Node.js 16:
    const result = Math.pow(3, 2); // 9

    // Korišćenjem Node.js 16, operator '**' se može direktno koristiti:*

    const result = 3 ** 2; // 9
Web Crypto API

Pre Node.js 16, ako ste želeli da obavljate kriptografske operacije, morali ste da koristite biblioteke (third-party) kao što su OpenSSL ili CriptoJS. Na primer, ako želite da kriptujete poruku pomoću AES enkripcije, morali biste da koristite biblioteku kao što je CriptoJS:

    // Bez ugrađene podrške za kriptografske operacije
    const crypto = require('crypto');

    function generateKey() {
    const key = crypto.randomBytes(32).toString('hex');
    console.log(key);
    }

    generateKey();

Sa Node.js 16, možete direktno da koristite Veb Cripto API:

    // Sa podrškom kriptografskih operacija
    const crypto = require('crypto');

    async function generateKey() {
    const algorithm = 'AES-GCM';
    const length = 256;
    const key = await crypto.generateKey(algorithm, {
        length,
    });
    console.log(key);
    }

    generateKey();

Node.js Version 18

Node.js verzija 18 objavljena je u aprilu 2022. i dolazi sa nekoliko novih feature-a i poboljšanja. Dva značajna feature-a su:

Async local storage

Asinhronizovana lokalna memorija je novi API koji omogućava programerima da čuvaju podatke koji su ograničeni na određeni kontekst izvršavanja, kao što je HTTP zahtev. Ovo olakšava deljenje podataka između funkcija koje se pozivaju u istom izvršnom kontekstu.

Pre Node.js 18, ako ste želeli da prenesete podatke između funkcija u lancu asinhronog poziva, morali ste da koristite argumente povratnog poziva (callback arguments) ili globalne promenljive. Na primer, ako želite da prosledite korisnički ID iz jedne asinhrone funkcije u drugu, morali biste da koristite globalnu promenljivu:

    let userId;

    function processRequest(request, callback) {
        userId = getUserIdFromRequest(request);
        processRequestData(request.data, callback);
    }

    function processRequestData(data, callback) {
        // koiršćenje userId
        callback();
    }

    function getUserIdFromRequest(request) {
        return request.headers['x-user-id'];
    }

Sa Node.js 18, možete koristiti Async Local Storage za prosleđivanje podataka između funkcija:

    const { AsyncLocalStorage } = require('async_hooks');

    const asyncLocalStorage = new AsyncLocalStorage();

    async function processRequest(request) {
    asyncLocalStorage.run(request, async () => {
        const userId = getUserIdFromRequest(request);
        await processRequestData(request.data, userId);
    });
    }

    async function processRequestData(data, userId) {
        // koiršćenje podataka i userId
    }
Stable Timers API

Pre Node.js 18, ugrađeni tajmeri ​​u Node.js su imali problem sa preciznošću koji je mogao da izazove njihovo pokretanje kasnije nego što se očekivalo. Ovo je bilo posebno problematično za aplikacije koje su se oslanjale na precizno merenje vremena, kao što su audio ili video obrada.
Sa Node.js 18, uveden je API za stabilne tajmere, koji tajmerima obezbeđuje zagarantovanu tačnost i dosledno ponašanje na različitim platformama. To znači da se možete osloniti na tajmere koji će aktivirati tačno kada očekujete, bez ikakvih varijacija.

    const { setStableTimeout } = require('timers');

    setStableTimeout(() => {
        console.log('Timeout fired!');
    }, 1000);

Ovaj kod će pokrenuti funkciju povratnog poziva tačno 1 sekundu nakon pokretanja tajmera, bez obzira na bilo kakve spoljne faktore koji mogu uticati na tajming, kao što je opterećenje sistema ili pomeranje sata. Ovo može biti posebno korisno za aplikacije u realnom vremenu koje zahtevaju precizno merenje vremena, kao što su razvoj igara ili multimedijalne aplikacije.

Nadogradnjom na Node.js 18 i korišćenjem API-ja za stabilne tajmere, možete osigurati da vaše aplikacije rade na najbolji mogući način i da pružaju besprekorno korisničko iskustvo.

Node.js Version 20

Node.js 20 predstavlja nadogradnju takozvane ‘major’ verzije, za koje se očekuje da će imati dugoročnu podršku (LTS) u oktobru 2024. Dolazi sa nekoliko optimizacija performansi i poboljšanja u vezi sa bezbednošću. Iako su još uvek u fazi razvoja, najavljene su velike promene i poboljšanja, zbog čega će nesumnjivo biti vredno čekanja.

Performance improvements

Node.js 20 ima nekoliko poboljšanja performansi koja ga čine bržim i efikasnijim. Ažuriranja uključuju brže vreme pokretanja, bolje upravljanje memorijom i poboljšano oslobađanje memorije (garbage collection).

Pre Node.js 20, ako ste želeli da otklonite i pronađete problem sa performansama ili gašenjem aplikacije, morali ste da koristite spoljne alate kao što su Chrome DevTools ili Node.js Inspector. Ovaj način bi mogao da oduzme mnogo vremena i zahteva postavku ovih alata koja je često komplikovana.

Sa Node.js 20, možete da koristite novi feature dijagnostičkih izveštaja da biste automatski generisali dijagnostičke izveštaje koji sadrže informacije o performansama vaše aplikacije, upotrebi memorije i drugim pokazateljima. Na primer, ovako bi izgledalo generisanje izveštaja:

    const { generateReport } = require('diagnostic_channel');

    generateReport('myapp', 'performance').then((report) => {
        console.log(report);
    });
Promise.any()

Pre Node.js 20, ako ste želeli da sačekate da se bilo koji Promise u nizu razreši ili odbije, morali ste da koristite spoljne biblioteke (third-party), kao što su Bluebird ili K. Na primer, ako ste imali niz Promise-a i želeli ste da sačekate da se prvi razreši, morali biste da koristite Bluebird na ovaj način:

    const Promise = require('bluebird');

    const promises = [promise1, promise2, promise3];

    Promise.any(promises).then((result) => {
        console.log(result);
    });

Sa Node.js 20, možete koristiti novu metodu Promise.any():

    const promises = [promise1, promise2, promise3];

    Promise.any(promises).then((result) => {
        console.log(result);
    });

Ovaj metod vraća Promise koji je razrešen i sadrži vrednost prvog razrešenog Promise-a u nizu. Ako su svi Promisi odbijeni, metoda se odbija sa AggregateError-om koji sadrži sve razloge odbijanja.
Metoda Promise.any(), pruža sažetiji i jednostavniji način za rukovanje višestrukim Promise-ima i čekanja da se prvi razreši ili odbije. Ovo može biti posebno korisno u situacijama kada treba da preuzmete podatke iz više izvora i želite da koristite onaj koji je najbrže odgovorio.

Nadogradnjom na Node.js 20 i korišćenjem metode Promise.any(), možete da pojednostavite svoj kod i učinite ga efikasnijim, bez oslanjanja na spoljne biblioteke.

Da biste instalirali najnoviju verziju Node.js, možete koristiti sledeću komandu u terminalu:

    nvm install 20.0.0

Ova komanda će preuzeti i instalirati najnoviju stabilnu verziju Node.js 20 na vašu mašinu.
